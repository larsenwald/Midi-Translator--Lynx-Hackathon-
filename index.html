MMT Scripting Reference
How scripts work

Each script is a plain Python file that runs on every incoming MIDI event. Variables for the current event are injected into the script's scope — you can read them, mutate them, or call helper functions to emit new messages. Scripts run top to bottom in the order they appear in the sidebar.

Scripts are stateless — no data persists between events. If you need a toggle or counter, that's not currently supported.

pitchintpitchwheelPitch bend value -8192 to +8191. Center (no bend) = 0
programintprogram_changeProgram number 0–127

Only read variables that exist for the current event_type, or you'll get a NameError. Always guard with an if event_type == check first.

Helper functions
FunctionDescription
send_note_on(note, velocity, channel=None)Emit a note on. channel defaults to incoming channel.
send_note_off(note, velocity=0, channel=None)Emit a note off.
send_cc(cc_num, value, channel=None)Emit a control change.
send_pitchbend(pitch, channel=None)Emit a pitch bend. pitch: -8192 to +8191.
send_program_change(program, channel=None)Emit a program change.
block()Suppress the original event entirely. Nothing is forwarded unless you also call a send_* helper.
Patterns
Mutate — remap a note
if event_type == 'note_on' and note == 60:
    note = 62  # C4 → D4
Replace — change event type entirely
if event_type == 'pitchwheel':
    send_cc(11, max(0, int(pitch / 8191 * 127)))
    block()  # suppress original pitch bend
Filter — block specific events
if event_type == 'note_on' and velocity < 20:
    block()  # drop ghost notes
Fan out — one event becomes many
if event_type == 'note_on':
    send_note_on(note, velocity, channel=1)
    send_note_on(note + 12, velocity, channel=2)
    block()  # replace original with both
Note map

Middle C is C4 = 60. Each octave is 12 semitones. Note name constants are available directly in scripts — note == C5 and note == 72 are equivalent. Sharps use Cs4 (C#4), flats use Db4.
